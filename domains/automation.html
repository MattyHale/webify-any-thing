Automation

Automation turns EO workflows from manual coordination into dependable operations.

What is Expected

Automation should handle repetitive and complex workflows without manual intervention. The platform is expected to automate ingestion, indexing, processing, and delivery once a trigger or request is received. All major functions must be accessible via APIs so users and third-party systems can integrate them into their own pipelines.

The platform should follow an event-driven design so new data automatically triggers downstream processing, state updates, and notifications.

Why It’s Important

Automation improves speed, scalability, and consistency. Manual workflows do not scale and introduce errors. Automated pipelines ensure repeatability and allow the platform to respond faster than human operators.

Key Automation Principles

API-First Design

Every capability available in the UI must also be available via APIs. The UI should consume the same APIs exposed to users.

Workflow Orchestration

Use orchestration to chain steps, manage retries, handle branching, and persist state in observable, re-runnable workflows.

Event-Driven Operations

Publish and react to events for data arrival, job state changes, and delivery to eliminate polling and reduce latency.

Remove Manual Touchpoints

Replace human checks with programmatic checks so operators handle exceptions, not the happy path.

User Automation

Provide CLIs, SDKs, and examples so customers can integrate the platform into their own automation stacks.
